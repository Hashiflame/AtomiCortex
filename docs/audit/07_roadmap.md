# 07 — Роадмап P0 → P3

> Принцип (AFML): сначала научиться честно измерять, потом убрать skew,
> потом улучшать сигнал, потом добирать risk-adjusted return портфельным слоем.
> Оценки времени — для одного разработчика с текущим знанием кодовой базы.
> Все «outcome» — честные диапазоны или «не измеримо заранее».

| Шаг | PR | Ожидание |

**Outcome P0:** не «метрики вырастут», а «числам можно верить». Реалистично
ожидать, что после дефляции текущий 4H-edge окажется в серой зоне (DSR 0.3–0.8);
если выше — уже есть основание для осторожного масштабирования.

нельзя; эффект на согласованность — почти детерминированно положительный.

## P2 — Качество сигнала (после того как P0-метрики покажут, что и где слабо)
- Перевалидировать пороги (quantile-based, код уже есть в validate_15m_models.py:187).
- Clustered feature selection на 45-фичевых моделях (whitelist-механика уже есть).
- live_enrichment (VPIN/OFI/liq): **сначала** материализовать исторически и в трейнинг,
  и в live одинаково (сейчас фичи не видит модель — #11); ожидания умерить:
  VPIN/OFI-эффекты документированы на тиковом/минутном горизонте (Easley-LdP-O'Hara 2012;
  Cont-Kukanov-Stoikov 2014), на 4H перенос не доказан — только эксперимент покажет.
- Meta-labeling: вторичная модель на ортогональных фичах (AFML ch.3.6) — только при
  ≥5k сигналов истории, иначе переобучение.
- ЯВНО НЕ ДЕЛАТЬ: смену класса модели (см. 04 §4.7), «добавить 50 фичей разом».

## P3 — Portfolio/risk (самый дешёвый прирост risk-adjusted return)
- Включить ETH/SOL в paper (сейчас торгуется только BTC — breadth=1; Carver:
  диверсификация по слабокоррелированным инструментам — главный множитель SR;
  IDM при 3 крипто-перпах с ρ≈0.8 скромный, ~1.1–1.2, но бесплатный).
- Vol-targeting позиции (σ-scaled sizing вместо фиксированного 1% risk).
- Подключить готовый regime-multiplier (A19) и заменить hardcoded vol-фильтр.
- Fractional Kelly (≤0.25; Thorp/MacLean-Ziemba) — только после P0-надёжного edge-оценивания.
